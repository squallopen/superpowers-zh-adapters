# v0.3.0 Claude Code / Codex 适配正式版

这版最重要的变化，是把 `Claude Code` 和 `Codex` 正式接进来了，同时把 vendored 上游 `obra/superpowers` 刷到了 `v5.0.6`。

其中 `Codex` 这部分，不是简单写一句“支持了”就算结束，而是把最容易出问题的 git 场景直接限制住，确保它能安全用。

## 相比 v0.2.2，重点改了什么

- 新增 `Claude Code` 安装与适配
- 新增 `Codex` 安装与适配
- vendored 上游 `obra/superpowers` 同步到 `v5.0.6`
- 首页、能力矩阵、安装说明和发布文档补齐了 `Claude Code` / `Codex` 文档入口
- 发布口径里明确写清楚：`Codex` 可用，但不是把上游默认动作硬搬过来

## 为什么说 Codex 现在可以安全用

关键不是“它和上游完全一样”，而是“不该硬跑的地方不会硬跑”。

这版对 `Codex` 主要加了两类硬限制：

- 如果当前已经在应用自己管理的 linked worktree / detached HEAD 里，就不要再盲目新建 worktree
- 如果当前环境不允许 `branch`、`push` 或创建 PR，就不要假装这些动作已经成功，而是改成 `commit + handoff`

直接说，就是：

- 能按 `Codex` 原生 skill / subagent 方式承接 `superpowers`
- 但不会在高风险 git 场景里冒充“和上游完全一样”
- 遇到限制时会老实降级，给出当前目录、提交结果和后续 handoff 信息

## Claude Code 这次补了什么

- 按官方 `skills/` + `CLAUDE.md` 结构安装
- 增加中文触发提示和中文文档输出约束
- 保留最接近上游原始 workflow 的使用方式

## 安装和更新层面这次补了什么

- `install-all.ps1`、`update-all.ps1`、`refresh-upstream-and-reinstall.ps1` 现在都接入了 `ClaudeCode` 和 `Codex`
- 新增 `install-claudecode.ps1`、`install-codex.ps1`
- 安装后会做一次轻量的 GitHub star 提示
- 仍然只官方维护 `Windows + PowerShell 7 + Git for Windows` 这条脚本链路

## 适合谁升级

- 你希望在 `Claude Code` 里直接用这套中文适配
- 你主要在 `Codex` 里工作，但又不想碰到 worktree / 分支收尾时的假兼容
- 你希望继续保留上游 `superpowers` 的 skill 结构，而不是换成另一套完全不同的中文实现

## 说明

- 这版包含 `Codex` 和 `Claude Code` 的项目级 smoke install 验证
- `Codex` 的安全边界已经写进适配层，不只是发布文案里口头说明
- GUI 层面仍然没有做每个工具的全量自动化回归，当前验证重点仍然是安装结果、模板生成和关键约束是否落地
