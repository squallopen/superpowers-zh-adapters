# v0.3.0 Claude Code / Codex 适配正式版

这版主要做了三件事：

- 正式支持 `Claude Code`
- 正式支持 `Codex`
- 把 vendored 上游 `obra/superpowers` 同步到 `v5.0.6`

`Codex` 这次不是停留在“能装上”这一层，而是把最容易踩坑的 git 场景先限制住了，所以现在可以更稳地用。

## 这次更新了什么

- `Claude Code` 现在按官方 `skills/` + `CLAUDE.md` 方式接入
- `Codex` 现在按 `.agents/skills` + `AGENTS.md` 方式接入
- 首页、安装说明、能力矩阵和工具文档都补齐了 `Claude Code` / `Codex` 的入口
- 发布口径也补上了 `Codex` 的安全边界，不再只是简单写一句“支持 Codex”

## 为什么说 Codex 现在更稳

- 如果当前已经在 linked worktree / detached HEAD 里，就不会再盲目新建 worktree
- 如果当前环境不允许 `branch`、`push` 或建 PR，就会老实走 `commit + handoff`
- 能按 `Codex` 原生 skill / subagent 方式承接 `superpowers` 的地方继续用，不能硬跑的地方就直接限住

## 适合谁升级

- 想在 `Claude Code` 里直接用这套中文适配
- 平时主要用 `Codex`，但不想在 worktree 或分支收尾上踩坑
- 想继续跟着上游 `superpowers` 走，而不是换一套完全不同的中文实现
