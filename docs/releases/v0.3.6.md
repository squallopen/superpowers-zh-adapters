# v0.3.6 同步上游 v6.0.3，并新增 ZCode 支持

这版主要做两件事：跟上 `obra/superpowers` 的 `v6.0.3`，以及把智谱 ZCode 加进本仓库的一键安装目标里。

## 更新了什么

- vendored 上游 `obra/superpowers` 已同步到 `v6.0.3`
- 新增 `ZCode` 目标，可以用 `install-all.ps1`、`update-all.ps1` 和 `refresh-upstream-and-reinstall.ps1` 安装或更新
- ZCode 的 User 模式会安装到 `~/.zcode/skills`
- ZCode 的 Project 模式会安装到 `<project>/.zcode/skills`
- ZCode 安装会追加中文触发词和关键 workflow 的 ZCode 适配说明
- README、能力矩阵、AI 安装说明和维护文档都补了 ZCode 入口

## 为什么要更新上游

`obra/superpowers` 的 6.x 是一次比较大的上游更新，重点包括：

- `subagent-driven-development` 的审查流程改成新的 task reviewer 结构
- 旧的 SDD scratch 位置不再放在 `.git/` 下面
- 多处说明从 Claude Code 专属说法改成更通用的 agent/harness 说法
- Codex、OpenCode 等 harness 的说明也有更新

这类变化如果不同步，中文适配层以后会越来越容易引用到旧文件名或旧流程。

## ZCode 怎么装

只装到当前用户：

```powershell
pwsh .\scripts\powershell\install-all.ps1 -Targets ZCode -Scope User
```

只装到当前项目：

```powershell
pwsh .\scripts\powershell\install-all.ps1 -Targets ZCode -Scope Project -ProjectRoot E:\path\to\project
```

如果你已经装过，只想更新：

```powershell
pwsh .\scripts\powershell\update-all.ps1 -Targets ZCode -Scope User
```

## 安全边界

ZCode 这次走本地 skill 目录安装，不会改 ZCode 官方插件缓存，也不会读写 credentials、token、provider 配置这类敏感文件。
